Step 2: Identify elements

On a basic level browsers interpret documents which are written in the HyperText Markup Language. Every element has a specific tag that we can use to locate it on a web page.

  • Selector Gadget
  • Developer Tools
  • Learn to understand HTML and CSS
# Load package
library(tidyverse); library(rvest)

# Select element
node <- read_html(url) %>%
  html_nodes(css = "#yw1 > table:nth-child(2)")

Further important topics: Document Object Model (DOM) and Cascading Style Sheets (CSS) (Munzert et al. 2015).

Generalizing the approach

Generalizing involves two steps:

  1. Wrapping our approach for a single target into a function
  2. Using for, apply or purrr::map to loop through pages

Application: Generalizing the approach

We need:

  1. A vector of URLs to loop over
  2. One function that scrapes the content
  3. One function that cleans the data

Firstly, we construct a vector that holds the URLs:

pages <- vector(mode = "character", length = 13) # Container
years <- seq(2005, 2017, 1)                      # "Dynamic element"

# Bring URL (static) and year (dynamic) together
for (i in seq_along(years)) {
  pages[[i]] <- paste0("https://www.transfermarkt.de/1-bundesliga/startseite/wettbewerb/L1/plus/?saison_id=", years[i])
}

Secondly, we wrap our former approach into a function:

# Wrap the code that worked for a single page into a function (scraper)
transfer_scraper <- function(x) {
  table <- read_html(x) %>%
    html_nodes(css = "#yw1 > table:nth-child(2)") %>%
    html_table(fill = TRUE, header = TRUE) %>%
    .[[1]]
}

Thirdly, the data cleaning should also be automated with a function:

clean_fun <- function(x) {
  # (1) Remove unused columns/rows
  table <- x[,c(-1, -9:-11)]
  table <- table[-1,]
  
  # (2) Rename columns
  colnames(table) <- c("team", "team.abb", "squad.size", "age.mean", "foreign.no", "mv.tot", "mv.mean")
  
  # (b) Apply the function
  table <- table %>%
    mutate_at(vars(mv.tot, mv.mean), mil_to_num)
}

Map over web sites

Using the map function, we can apply our functions to the URLs.

# Basically, very short for-loops:
# (1) Scrape transfermarkt.de
multi <- map(pages, ~transfer_scraper(.))

# (2) Clean the data
multi.clean <- map(multi, ~clean_fun(.))

RSelenium

Basically, we just steer a browser from within R (Chrome, PhantomJS). The code snippet below shows how we can navigate Chrome to the FAZ archive and search for âDeutsches Institut fÃ¼r Wirtschaftsforschungâ.

# Navigate to FAZ archive

# Identify the search box

# Send search terms

# Identify the search button and click it

# Click
search.button$clickElement()

Some notes

Please, keep in mind (Ethics of Web Scraping):

  • Use Application Programming Interfaces whenever possible (ask for the data)
  • Do not send too many requests at once (DoS attack)
  • Sys.sleep(sample(seq(0,2,0.5), 1)) at the end of each loop

Learn to build proper loops and regex (Wickham & Grolemund 2017).
